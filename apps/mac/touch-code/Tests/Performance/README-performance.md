# Performance baseline + tests — placeholder for 0005 M8

Reserved for `GitViewerPerformanceTests.swift` and companion artefacts. Populated in M8.

## Expected contents

- `GitViewerPerformanceTests.swift` — the performance test file (writes to `baseline.json`
  in measurement mode; asserts against it in CI mode).
- `baseline.json` — the per-machine baseline (see format below). **Not** checked into the
  repository's main history: it is regenerated by each contributor on first local run and
  by CI on its own runner. The file is gitignored (entry added here when M8 lands).
- `fixtures/` — synthetic diff / log fixtures (e.g. `diff-1000-lines.txt`, a generated
  1 000-line unified diff, checked in so measurements are deterministic).

## Baseline format (refined in 0005 plan M8)

```json
{
  "version": 1,
  "machineKey": "arm64-apple-macos-M1",
  "capturedAt": "2026-04-20T10:00:00Z",
  "samples": { "parse_ms": { "p50": 14.2, "p95": 38.1, "max": 55.8 },
               "reducer_ms": { "p50": 0.8, "p95": 1.9, "max": 3.1 } }
}
```

Assertion threshold:

    assertion_ceiling = max(designBudget, baseline.p95 × driftMargin)

with `driftMargin = 1.25` and `designBudget.parse_ms = 80`, `designBudget.reducer_ms = 20`.

## Running the baseline collection

See `0005 M8` concrete-steps for the exact commands. The short version:

1. Gate via `TC_RUN_PERFORMANCE_TESTS=1`.
2. Run 50 samples end-to-end; discard the first 5 as warm-up.
3. Write `baseline.json` with median / P95 / max per metric.
4. On subsequent runs, the test asserts against the ceiling formula above.
